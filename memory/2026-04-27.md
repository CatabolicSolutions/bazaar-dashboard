### Adaptive volatility trigger (dynamic trigger refinement)
- Problem: fixed 0.2% misses bigger rips, fixed 0.5% misses micro-swings

- Quiet market (0.05% avg) → 0.15% trigger (tight)
- Normal market (0.10% avg) → 0.20% trigger
- Volatile market (0.30% avg) → 0.50% trigger (wide, let runs extend)

### Conor's P&L concern: "$13.34 is fake"
- Wallet balance: 0.110342 WETH @ $2,289.41 = **$253.15 total**
- Unrealized P&L: **+$0.66** (from entry $2,283.45)
- Bot reported P&L: **$13.34** — completely wrong
- Dashboard P&L of $13.34 shown but no actual sell ever executed

### Root cause: P&L added before swap confirmation + 1inch rounding error
- **P&L bug** (`bot/main.py` line 261-262): `flip_pnl = change_pct / 100 * bal['weth_usd']` and `state['total_pnl'] += flip_pnl` are executed **BEFORE** `do_swap()` runs. If the swap fails (which it does), the P&L was already added — phantom profit.
- **1inch swap rounding error**: `int(bal['weth'] * 1e18)` can produce a value 8 wei higher than actual on-chain balance. 1inch returns "Not enough balance" because of this 8 wei rounding difference.
- **Re-fire loop**: On the next tick, the sell condition still fires (change_pct still >= trigger, no cooldown since state wasn't saved). Each fire adds another flip_pnl to total_pnl without any actual trade.
- **startup log evidence** (04:43:30 UTC): Bot tried to SELL with trigger=0.22%, change=+0.24%, 1inch returned 400 error "Not enough balance" — difference was only 8 wei.
- Multiple re-fire attempts compounded the phantom P&L from $1.12 (real) to $13.34 (fake).

### Adaptive trigger stuck at 0.5% cap
- Deque initialized with 30 entries from the current run (all ~0.23-0.29%)
- Price maxed at +0.46% from entry $2,283.45
- Trigger at 0.5% cap due to deque averaging higher volatility from early entries
- Price never exceeded cap → no sell condition met after the failed attempt
- **Core issue**: with only ~$250 capital and 0.2% scalp threshold, the 0.5% trigger cap is too wide for the bot to fire consistently. The deque's average plus multiplier produces a trigger that almost always exceeds the actual price movement.

### Fixes needed (identified, not yet deployed)
1. **Move P&L calculation inside do_swap(), after swap succeeds** — prevents phantom P&L from failed swaps
2. **Subtract 1000 wei buffer from swap amount** — prevents the 8 wei rounding error ("not enough balance")
3. **Add cooldown after failed swaps** — prevent re-fire loop
4. **Lower trigger cap from 0.5% to 0.3%** — with $250 capital, 0.2% scalp threshold, the cap shouldn't exceed 0.3% for reasonable trade frequency
5. **State file shows total_pnl=1.12 on disk** (real P&L from initial CBBTC→WETH swap at 03:08 UTC), but bot status file shows 13.34 (in-memory phantom). State file is only saved when a trade executes — need to also save state periodically.
