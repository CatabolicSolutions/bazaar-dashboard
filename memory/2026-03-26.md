## Operating rule — anti-drift update format
- In the Discord collaboration thread, every progress update must begin with exactly one of:
  - `DONE`
  - `BLOCKED`
  - `MISSED`
- Every update must also include all of:
  - commit hash or exact blocker
  - files changed
  - tests passed/failed
  - next immediate step
- If an update lacks those fields, it does not count.
- If two milestone deadlines are missed in a row, autonomy narrows and work should be assigned one small slice at a time with proof required before the next slice.
- This rule exists because drift has been an execution-discipline failure, not a technical blocker.
