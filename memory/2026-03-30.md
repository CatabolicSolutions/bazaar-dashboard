### The Critical Gap

**Current Flow (Dead End):**
```
Dashboard Click → Local JSON file (queue) → [NO FURTHER ACTION]
```

**Required Flow for Live Trading:**
```
Dashboard Click → ExecutionIntent → Risk Check → Preview → Approval → Place Order → Record
```

**Missing Bridge:**
- Dashboard queue entries are simple dicts
- Execution Service expects `ExecutionIntent` dataclass objects
- No automated pickup of queued items
- No trigger to move from "queued" state to "preview/execute"

### Missing Pieces Blocking Live Orders

1. **Intent Translation Layer**
   - Convert dashboard leader JSON → ExecutionIntent
   - Map dashboard fields to intent fields

2. **Execution Trigger**
   - New dashboard action: "Execute Now"
   - API endpoint to handle immediate execution flow

3. **Preview/Approval UI**
   - Show broker preview before placing order
   - User confirmation step
   - Error handling display

4. **Order State Persistence**
   - Track order from placement through fill
   - Reconcile broker state with local state

### Minimal Slice to Enable First Trade

**Option A: Immediate Execution (Fastest)**
1. Add "Execute Now" button to dashboard
2. Create bridge: `leader → ExecutionIntent`
3. Immediate flow: preview → user confirm → place order
4. Skip the queue complexity for now

**Option B: Queue-Based Execution (More Complete)**
1. Background worker polls `execution_queue.json`
2. Converts queue items to intents
3. Runs full state machine
4. Requires persistent process

**Recommendation: Option A**
- Gets you trading today
- Can add queue sophistication later
- Matches your "execution-focused" requirement
