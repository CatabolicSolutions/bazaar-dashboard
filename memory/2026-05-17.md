# 2026-05-17 - Uniswap Rotator Swap Debug Complete

## Problem
All swap attempts on Base chain were reverting — both single-hop and multi-hop — regardless of router address or function signature used.

## Timeline
1. Started with V3 Router01 `0xE59242...` — `exactInputSingle` returned empty (proxy, no implementation)
2. Tried `SwapRouter02` `0x2626664c2603336E57B271c5C0b26F421741e481`:
   - `exactInputSingle` 7-param (no deadline) ✅ worked via eth_call
   - `exactInputSingle` 8-param (with deadline) ❌ reverted
   - `exactInput` (multi-hop path) ❌ hit `InvalidFEOpcode` (function absent)
   - Multicall chaining two singles with recipient=ROUTER ❌ STF (SafeTransferFrom) — router got WETH, wallet couldn't supply second leg
3. Tried `UniversalRouter` `0x3fC91A3a...` — `execute()` consistently reverted with `0xd81b2f2e` (internal V3 swap error)
4. **Found solution**: Multicall chaining two `exactInputSingle` with recipient=WALLET for both legs
   - Leg 1: takes USDC from wallet → sends WETH to wallet
   - Leg 2: takes WETH from wallet (router has infinite approval) → sends CBBTC to wallet
   - Verified: nonce 487, tx `b745e94ec7a8c89387ea9e9ce82582c101965f7cb114019b5aaa144d8a2aee5c`, status=1 ✅

## Durable Truths - Base SwapRouter02
- **Correct address**: `0x2626664c2603336E57B271c5C0b26F421741e481`
- **exactInputSingle struct**: 7 params, NO deadline — `(address tokenIn, address tokenOut, uint24 fee, address recipient, uint256 amountIn, uint256 amountOutMinimum, uint160 sqrtPriceLimitX96)` — selector `04e45aaf`
- **exactInput NOT available** on this SwapRouter02 version on Base — do not use multi-hop paths
- **Multi-hop via multicall**: chain two `exactInputSingle` with both recipients set to wallet. Works because router has infinite approval for both USDC and WETH.
- **UniversalRouter** (`0x3fC91A3a...`) also not functioning for V3 swaps on Base
- **Wallet**: `0xFf9cc57be71A7b851A422a851600ac23B3843e27` has max approvals for SwapRouter02 on both USDC and WETH
- **QuoterV2**: `0x3d4e44Eb1374240CE5F1B871ab261CD16335B76a` — works with `quoteExactInputSingle` tuple API
- **V3 Factory**: `0x33128a8fC17869897dcE68Ed026d694621f6FDfD`
